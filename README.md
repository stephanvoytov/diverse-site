# Diverse — Лендинг франшизы в России

Официальный лендинг представителя бренда **Diverse** в России (ООО «Хаус»).

Сайт: [diversebrand.ru](https://diversebrand.ru)

---

## О проекте

Лендинг франшизы магазинов одежды Diverse. Бренд основан в 1993 году в Гданьске (Польша), насчитывает 400+ магазинов в 9 странах. ООО «Хаус» (Калининград) — эксклюзивный дистрибьютор в РФ и СНГ.

### Страницы

| Маршрут | Контент |
|---------|---------|
| `/` | Главная: герой, бренд в цифрах, матрица, франшиза, карта, контакты |
| `/about` | История Diverse, философия, 3 линии (Diverse / DEXT / Coalition), таймлайн |
| `/collection` | Коллекции бренда: full-screen параллакс-секции с описаниями |
| `/franchise` | Условия франшизы: 3 варианта, сравнение, преимущества, форма |
| `/stores` | Магазины: интерактивная карта + фото |

## Переменные окружения (.env.local)

### SMTP (основной канал заявок)

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `SMTP_HOST` | да | SMTP-сервер (напр. `smtp.yandex.ru`) |
| `SMTP_PORT` | нет | По умолчанию `465` |
| `SMTP_SECURE` | нет | `false` — отключить TLS (по умолчанию включён) |
| `SMTP_USER` | да | Логин от почтового ящика |
| `SMTP_PASS` | да | Пароль от почтового ящика |
| `MAIL_TO` | да | На какой адрес приходят заявки |

### Telegram (второй канал — дублирует все заявки)

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `TELEGRAM_BOT_TOKEN` | да | Токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | да | ID чата (админ или группа), куда бот шлёт заявки |

**Как получить Telegram-бота:**
1. В Telegram → **@BotFather** → `/newbot` → получаете токен (это `TELEGRAM_BOT_TOKEN`)
2. Пишете боту любое сообщение (он пока ничего не ответит)
3. Открываете в браузере `https://api.telegram.org/bot<TOKEN>/getUpdates` — в ответе найдёте `"chat":{"id":123456789}` — это `TELEGRAM_CHAT_ID`

### Прочие

| Переменная | Описание |
|------------|----------|
| `NEXT_PUBLIC_BASE_PATH` | Базовый путь (если сайт не в корне домена) |
| `NEXT_PUBLIC_FORM_ENDPOINT` | Куда формы шлют POST (по умолч. `/api/lead`) |

## Как работают заявки

1. Посетитель заполняет форму — `POST /api/lead`
2. Сервер одновременно отправляет:
   - **SMTP** — письмо на почту (`MAIL_TO`)
   - **Telegram** — сообщение в чат администратора
3. Если хотя бы один канал сработал — форма показывает успех
4. Если оба упали — форма показывает ошибку, заявка сохраняется в `localStorage` у пользователя

Таким образом, заявка не теряется, даже если SMTP-сервер временно недоступен.

### Формы на сайте

- **3-шаговая модальная** — на главной (формат → город → имя/телефон), прогресс-бар
- **FranchiseContent** — на странице `/franchise`
- **Contacts** — в блоке контактов на главной
- **CallbackWidget** — плавающий виджет обратного звонка

## Деплой

Сайт использует Next.js API-роуты (`/api/lead`), поэтому **не экспортируется в статику**. Требуется Node.js-хостинг, поддерживающий Next.js (Vercel, Railway, собственный сервер).

```bash
npm run build
npm run start
```
